Security & Compliance
How We Handle HIPAA

HIPAA compliance isn't a feature — it's infrastructure. These are the non-negotiable pieces built into the architecture from day one, not added later.

🔒
Encryption at Rest & In Transit
All data stored in the database is encrypted. All data moving between the user's browser and the server travels over HTTPS with TLS. No plaintext data anywhere in the system.
📋
Audit Logs
Every action a user takes — login, assessment started, result viewed, data changed — is recorded in an immutable audit log. HIPAA requires this. It also protects Jessica legally.
🏥
BAA with AWS
AWS offers HIPAA-eligible services and will sign a Business Associate Agreement (BAA). This is a legal requirement if the app touches Protected Health Information (PHI). We set this up before writing a single line of code.
👤
Access Controls & MFA
Users only see their own data. Admins have separate, controlled access. Multi-factor authentication required for all accounts. Role-based permissions built into the backend.
🔐
Secure Auth (Auth0 / Cognito)
We use an industry-standard identity provider — not a home-built login system. These services handle token management, session expiry, and brute-force protection out of the box.
⚖️
HIPAA Attorney Review Recommended
We build to technical HIPAA standards, but Jessica should have a HIPAA attorney review the final product and policies (privacy policy, terms of service, BAA templates) before launch. This is her legal protection.
Technology
The Tech Stack — And Why

Every choice here is deliberate — built for security, scalability, and maintainability. Nothing exotic, nothing that creates vendor lock-in.

Frontend
React.js + Tailwind CSS
React is the industry standard for web apps. It handles complex, multi-step flows (like the assessment wizard) cleanly and is easy to maintain and expand. Tailwind keeps the UI consistent and fast to build.
Backend
Python / FastAPI
Python is dominant in healthcare-adjacent tech. FastAPI is fast, modern, and has strong data validation built in — critical when you need to trust the inputs going into Jessica's decision engine.
Database
PostgreSQL
Reliable, battle-tested, and handles the relational data this app needs — users, assessments, results, audit logs — all linked together correctly. Supports encryption at rest.
Cloud Hosting
Amazon Web Services (AWS)
AWS is one of the few cloud providers that offers HIPAA-eligible infrastructure and will sign a BAA. This is not optional — it's a compliance requirement. Also scales easily as user volume grows.
Authentication
Auth0 or AWS Cognito
Battle-tested identity management. Handles MFA, secure sessions, password resets, and role-based access. Far safer than building a login system from scratch.
Payments (if needed)
Stripe
If this becomes a SaaS product with subscriptions, Stripe handles billing, invoicing, and subscription management cleanly. Integrates with the backend in a few days of work.
